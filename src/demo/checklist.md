# Критерии оценки ДЭ

## База данных

#### Проектирование базы данных (ERD)

2 - Все основные сущности определены, все атрибуты (поля) определены, идентификатор в каждой таблице присутствует, созданы связи между сущностями, названия таблиц и полей в едином стиле, отождествляются с сущностями и атрибутами предметной области, разработанная база данных находится в 3НФ

#### Реализация базы данных в конкретной системе управления базами данных

2 - Созданы таблицы, необходимые для разрабатываемой подсистемы и реализуемого функционала, созданы первичные ключи, созданы связи между таблицами Определены типы данных полей

#### Импортирование данных. Создание скрипта БД

2 - Все данные объектов загружены верно и в правильном формате, скрипт содержит описание структуры и хранящиеся данные

## Общие стили и оформление кода приложения

#### Формирование алгоритмов разработки программных модулей в соответствии с техническим заданием

3– Алгоритм оформлен в соответствии ГОСТ и содержит все основные шаги алгоритма

#### Разработка системы в соответствии с руководством по стилю

2 - Приложение соответствует руководству по стилю (шрифт, цвет), на окне приложения присутствует заголовок, размещен логотип на главной форме

#### Разработка интерфейса программного модуля

2 - Разработан последовательный пользовательский интерфейс: есть возможность перемещаться между существующими окнами/страницами в приложении (в том числе обратно, например, с помощью кнопки «Назад»)

#### Оформление исходного кода приложения в соответствии с соглашением об именовании и стилем CamelCase 

1 - Идентификаторы соответствуют стилю CamelCase, смысл именования очевиден, использовано не более одной команды в строке

#### Использование комментариев по коду

1 - Комментарии дают полное и очевидное понимание кода

#### Создание сообщений обратной связи с пользователем

1 - Сообщения полезные и информативные, содержат полную информацию о совершенных ошибках пользователя и порядок дальнейших действий 
2 - Присутствуют визуальные подсказки для пользователя при вводе данных, сообщения о совершенных ошибках пользователя и подсказки для их исправления 

#### Разработка модуля для проверки функциональности 

2 – Приложение корректно работает и не происходит аварийного завершения работы и создан документ docx и вложены скриншоты корректной работы системы


## Разработка экранных форм (окно/страница) 

### Список объектов

#### Информация об объекте

4 - Список объектов отображается в соответствии с макетом, выводится информация об объекте (тип, наименование и т.д. в соответствии с заданием) из БД 

#### Корректное отображение фотографии 

2 – Фотография отображается и отображение всех фотографий приведены к единому виду (присутствует заглушка) 

#### Сортировка списка

2 – Присутствует возможность сортировки (по возрастанию и убыванию) по количеству на складе

#### Фильтрация списка

2 – Присутствует возможность фильтрации по производителю, первым элементом в выпадающем списке должен быть «Все производители»

#### Поиск в списке

2 – Присутствует возможность поиска по по всем доступным данным с текстовым типом

#### Возможность поиска, сортировки, фильтрации в реальном времени

2 – Поиск, сортировка и фильтрация должны происходить в реальном времени

### Добавление/редактирование объекта: 

#### Загрузка данных из БД в поля для ввода

2 - При редактировании объекта в поля для ввода загружены данные из БД

#### Реализация переходов на окно/страницу добавления/редактирования объекта

1 - Реализованы переходы на окно добавления/редактирования выбранного объекта

#### Проверка значений, вводимых данных

1 - Реализованы проверки значений всех вводимых данных объекта, указанных в задании

#### Сохранение данных в БД

4 - Данные при добавлении сохраняются в БД, данные при редактировании изменяются в БД

#### Поля для заполнения 

2 - Присутствуют все поля для заполнения, указанные в задании

#### Обновление данных в окне списка объектов

1 - После добавления и редактирования объекта данные в окне списка объектов обновляются

### Удаление товара

#### Разработка возможности удаления товара администратором

2 – Присутствует возможность удаления товара из списка товаров и БД, если такого товара нет в заказах
